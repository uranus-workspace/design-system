title: AppShell
description: A casca completa de uma aplicação dashboard - sidebar, header, conteúdo e painel lateral opcional.
---

`AppShell` orquestra `SidebarProvider`, `SidebarInset` e os slots de header e painel direito. A API **composta** (`AppShell.Sidebar`, `AppShell.Inset`, `AppShell.Header`, `AppShell.Content`, `AppShell.RightPanel`) é a forma recomendada. A API legada (`sidebar`, `header`, `rightPanel` + `children` como corpo) permanece para compatibilidade.

A área principal fica em `AppShell.Content` (ou em `children` no modo legado), com **rolagem vertical** (`overflow-y-auto`) para o header fixo não competir com o scroll da página.

## Uso (composto)

```tsx
import { AppHeader, AppShell, AppSidebar } from '@uranus-workspace/blocks';

<AppShell defaultSidebarOpen>
  <AppShell.Sidebar>
    <AppSidebar groups={navigationGroups} logo={<Logo />} />
  </AppShell.Sidebar>
  <AppShell.Inset>
    <AppShell.Header>
      <AppHeader breadcrumbs={<Breadcrumbs />} userMenu={<UserMenu />} />
    </AppShell.Header>
    <AppShell.Content>{children}</AppShell.Content>
  </AppShell.Inset>
  <AppShell.RightPanel>{/* opcional: DetailDrawer */}</AppShell.RightPanel>
</AppShell>
```

## Uso (legado)

```tsx
import { AppShell } from '@uranus-workspace/blocks';

<AppShell
  sidebar={<AppSidebar groups={navigationGroups} logo={<Logo />} />}
  header={<AppHeader breadcrumbs={<Breadcrumbs />} userMenu={<UserMenu />} />}
>
  {children}
</AppShell>
```

## Anatomia

**Composto**

- `AppShell.Sidebar` — navegação primária (fora do `<main>`).
- `AppShell.Inset` — região principal ao lado da sidebar (usa `SidebarInset` / `<main>`).
- `AppShell.Header` — opcional, dentro do inset (sticky quando o conteúdo é `AppHeader`).
- `AppShell.Content` — corpo rolável da página.
- `AppShell.RightPanel` — opcional, irmão do inset (painel direito).

**Legado**

- `sidebar`, `header`, `rightPanel`, `children` — mesmo papel que os slots acima.

## Diretrizes

- Use **um** `AppShell` por aplicação. O bloco assume que a navegação é única no nível raiz.
- O bloco **não** define um `<h1>`. O `<h1>` da tela mora no `PageHeader`.
- Para autenticação use `AuthLayout`, não `AppShell`.

## Estado controlado vs. não controlado

```tsx
<AppShell defaultSidebarOpen={false}>
  <AppShell.Sidebar>...</AppShell.Sidebar>
  ...
</AppShell>
```

Por padrão o estado da sidebar é persistido em cookie pelo `SidebarProvider` do `@uranus-workspace/design-system`. Para controlar via app state passe `sidebarOpen` + `onSidebarOpenChange`.
