---
title: Blocks
description: Composite layouts built by combining core primitives.
---

Blocks live in `@uranus-workspace/blocks` and are opinionated compositions of the primitives in `@uranus-workspace/design-system`. They exist so product teams don't re-implement the same `PageHeader` or `EmptyState` across five apps with five different spacings.

A block is the right abstraction when:

- Multiple products need the same layout shape.
- The layout has an accessibility contract worth centralizing (landmarks, heading order).
- It composes **only** existing primitives — a block never redefines a button or an input.
