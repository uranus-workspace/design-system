`NotificationList` é a lista que vive dentro do `Popover` do sino no `AppHeader`. A composição é explícita com **`NotificationList.Header`**, **`NotificationList.List`** e **`NotificationList.Item`**. Itens com `unread` mostram um ponto azul e marcam `aria-current="true"`.

## Uso

```tsx
import { NotificationList } from '@uranus-workspace/blocks';
import { Popover, PopoverContent, PopoverTrigger, Button } from '@uranus-workspace/design-system';
import { Bell } from 'lucide-react';

<Popover>
  <PopoverTrigger asChild>
    <Button variant="ghost" size="icon" aria-label="Notificações">
      <Bell aria-hidden className="size-4" />
    </Button>
  </PopoverTrigger>
  <PopoverContent className="w-80 p-0" align="end">
    <NotificationList>
      <NotificationList.Header
        title="Notificações"
        unreadCount={2}
        onMarkAllRead={markAllRead}
      />
      <NotificationList.List>
        <NotificationList.Item
          title="Novo comentário em Apollo"
          description="Bruno: ficou ótimo!"
          timestamp="há 2 minutos"
          unread
          onSelect={() => router.push('/comments/1')}
        />
        <NotificationList.Item
          title="Build concluído"
          description="main passou em 2m31s"
          timestamp="há 1 hora"
        />
      </NotificationList.List>
    </NotificationList>
  </PopoverContent>
</Popover>
```

## Subcomponentes

- **`NotificationList.Header`** — header com `title?`, `unreadCount?`, `onMarkAllRead?`. O botão "Marcar tudo como lido" só aparece quando `unreadCount > 0`.
- **`NotificationList.List`** — `<ul>` rolável (`max-h-96 overflow-auto`).
- **`NotificationList.Item`** — `<li>` com `title`, `description?`, `timestamp`, `unread?`, `onSelect?` (vira botão clicável).
