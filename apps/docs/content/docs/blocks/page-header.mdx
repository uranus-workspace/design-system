---
title: PageHeader
description: The top-of-page banner that names a view and surfaces its primary actions.
---

`PageHeader` renders as a `<header>` landmark (`role="banner"` when used at the top of a view) containing an `<h1>`, an optional description, optional breadcrumbs, and a slot for actions.

## Usage

```tsx
import { PageHeader } from '@uranus-workspace/blocks';
import { Button } from '@uranus-workspace/design-system';

<PageHeader
  title="Projects"
  description="Manage every project inside your Uranus workspace."
  actions={
    <>
      <Button variant="outline">Import</Button>
      <Button>New project</Button>
    </>
  }
/>
```

## Guidance

- Each view has **one** `PageHeader` and **one** `<h1>`. Don't nest PageHeaders.
- Put navigation context (breadcrumbs) in the `breadcrumbs` slot, not as part of the title.
- Keep actions to at most two visible; overflow into a menu when there are more.
