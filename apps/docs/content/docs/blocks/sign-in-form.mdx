`SignInForm` valida email, senha e “lembrar” com **Zod** (`signInFormSchema`) e **`react-hook-form`**, usando os primitivos `Form` / `FormField` de `@uranus-workspace/design-system`. O `onSubmit` recebe valores já validados no cliente; conecte ao seu backend ou provedor de auth.

Para **SSO apenas** (produto sem email/senha), use `credentials="hidden"` e passe botões em `socialProviders` (veja `OAuthProviderButton`). Para fluxos híbridos, coloque `OAuthProviderButton` seguido de `AuthDivider` dentro de `socialProviders` — o bloco de credenciais aparece abaixo.

## Uso

```tsx
import { SignInForm, signInFormSchema } from '@uranus-workspace/blocks';
import type { SignInFormValues } from '@uranus-workspace/blocks';

<SignInForm
  forgotPasswordHref="/forgot-password"
  signUpHref="/sign-up"
  loading={isPending}
  error={error}
  onSubmit={async (values: SignInFormValues) => {
    await signInAction(values);
  }}
/>
```

## Schema (exportável)

Importe **`signInFormSchema`** ou infira **`SignInFormValues`** via `import type { z } from 'zod'` e `z.infer<typeof signInFormSchema>` — útil para validar o mesmo contrato na server action ou na API sem duplicar regras.

```ts
signInFormSchema.safeParse(await request.json());
```

## Somente SSO

```tsx
import { OAuthProviderButton, SignInForm } from '@uranus-workspace/blocks';

<SignInForm
  credentials="hidden"
  signUpHref="/sign-up"
  socialProviders={
    <>
      <OAuthProviderButton provider="google" onClick={() => signInWith('google')}>
        Continue com Google
      </OAuthProviderButton>
    </>
  }
/>
```
Neste modo **não** é necessário passar `onSubmit` de credenciais.

## Slots

- `socialProviders` — botões SSO acima das credenciais (quando `credentials='visible'`).
- `credentials` — `'visible'` (padrão) ou `'hidden'` para telas só OAuth.
- `forgotPasswordHref` / `signUpHref` — ocultos quando omitidos.

## Acessibilidade

- Headings começam com `<h1>`. O `AuthLayout` não renderiza heading próprio.
- Erros do servidor como `Alert role="alert"`; erros de campo vêm de `FormMessage`.
- `autoComplete` adequado e `required` nativo para reforço do browser.
