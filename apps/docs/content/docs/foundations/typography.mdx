The Uranus 2026 brand manual pairs two typefaces:

- **Lexend Exa** — primary / display. Selected for its wide horizontal amplitude ("imensidão do horizonte espacial"). Used for headings, brand moments, and numerical displays.
- **Poppins** — secondary / body. A geometric sans-serif that balances sophistication and utility across dense UI.

Both families are loaded via Google Fonts. In Next.js apps we use `next/font/google` to inline them; in Storybook they come from a CDN link in `preview-head.html`.

## Families

| Role | Token | Stack |
| --- | --- | --- |
| Display | `font-display` | `Lexend Exa, Poppins, system-ui` |
| Body | `font-sans` | `Poppins, system-ui` |
| Mono | `font-mono` | `ui-monospace, SF Mono, Menlo…` |

```tsx
<h1 className="font-display text-5xl tracking-tight">Uranus</h1>
<p className="font-sans text-base text-muted-foreground">
  Um hub de serviços flexível e escalável.
</p>
```

## Weights

Lexend Exa: `300`, `400`, `500`, `600`, `700`, `800`.
Poppins: `300`, `400`, `500`, `600`, `700`.

Reference via the Tailwind classes `font-light`, `font-normal`, `font-medium`, `font-semibold`, `font-bold`, `font-extrabold`.

## Letter spacing

Display headings default to `-0.02em` (tight) for a disciplined, technical tone. For brand moments where Lexend Exa takes center stage, allow `-0.04em` (tighter) to respect the wide counterforms.

## Do / Don't

- **Do** pair display Lexend Exa headings with Poppins body for long-form content.
- **Do** use `font-display` for numeric displays (dashboards, KPIs) to reinforce the brand.
- **Don't** mix Lexend Exa with a third display typeface. The pairing is intentional.
- **Don't** use all-caps Lexend Exa at small sizes — legibility suffers below 14px.
