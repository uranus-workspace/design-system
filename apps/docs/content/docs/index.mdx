The Uranus Design System is the single source of truth for how Uranus Technologies products look, feel, and behave. It is a living system: documentation, code, and design artifacts evolve together.

## What's inside

- **Foundations** — the visual and behavioral primitives (color, typography, spacing, motion, accessibility, voice) that every interface shares.
- **Components** — focused, reusable UI primitives (`@uranus-workspace/design-system`) built on shadcn/ui, Tailwind CSS v4, and Motion.
- **Blocks** — higher-level composite layouts (`@uranus-workspace/blocks`) assembled from primitives.
- **Patterns** — opinionated guidance for recurring flows like forms, empty states, and navigation.

## Inspiration

The structure and depth of this documentation draws from systems we admire:

- [Carbon Design System](https://carbondesignsystem.com/) — IBM
- [Olist Design System](https://designsystem.olist.io/)
- [Apple Human Interface Guidelines](https://developer.apple.com/design/)
- [Microsoft Fluent 2](https://fluent2.microsoft.design/)
- [Samsung One UI](https://developer.samsung.com/one-ui/index.html)
- [Wise Design](https://wise.design/design-at-wise)

Visual identity — color, typography, motion personality — comes from the Uranus branding manual, not from these references.
