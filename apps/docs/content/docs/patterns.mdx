---
title: Padrões
description: Guia opinativo para fluxos recorrentes — formulários, empty states, navegação, exibição de dados, feedback.
---

Padrões são a forma como resolvemos problemas de UX recorrentes de maneira consistente. Diferente de componentes, um padrão pode não corresponder a uma única peça de código exportada — é um playbook que diz quais componentes combinar, em que ordem, com qual texto.

Esta seção vai crescer conforme lançamos produtos sobre o sistema. Entradas iniciais previstas:

- **Formulários** — posicionamento de rótulos, validação, sumários de erro, progressão multi-step.
- **Empty states** — quando mostrar o quê, tom de voz, hierarquia de ação.
- **Navegação** — barra superior, sidebar, breadcrumb, uso de tabs.
- **Exibição de dados** — tabelas, listas, cards, paginação, trade-offs de scroll infinito.
- **Feedback** — toasts, diálogos, banners, mensagens inline.
