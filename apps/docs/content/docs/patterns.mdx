---
title: Patterns
description: Opinionated guidance for recurring flows — forms, empty states, navigation, data display, feedback.
---

Patterns are how we solve recurring UX problems consistently. Unlike components, a pattern might not correspond to a single exported piece of code — it's a playbook that tells you which components to combine, in what order, with what copy.

This section will grow as we ship products on the system. Initial entries planned:

- **Forms** — label placement, validation, error summaries, multi-step progression.
- **Empty states** — when to show what, tone of voice, action hierarchy.
- **Navigation** — top bar, side bar, breadcrumb, tab usage.
- **Data display** — tables, lists, cards, pagination, infinite scroll trade-offs.
- **Feedback** — toasts, dialogs, banners, inline messages.
