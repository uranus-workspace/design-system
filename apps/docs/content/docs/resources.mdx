## Branding manual

The **Uranus Manual de Marca 2026** is the source of truth for visual identity. Every token in `@uranus-workspace/tokens` traces back to a value in the manual — the cosmic palette (Azul Profundo, Azul Marinho, Azul Turquesa, Lilás Claro) and the Lexend Exa / Poppins type pairing live in `packages/tokens/src/colors.ts` and `typography.ts`.

The live brand site is [uranus.com.br](https://uranus.com.br/). When the brand team updates the manual, values flow one direction: `packages/tokens/src/*.ts` → generated CSS → every app and component.

## Changelog

Releases are driven by [Changesets](https://github.com/changesets/changesets). Add a changeset with `pnpm changeset` before merging any PR that touches `@uranus-workspace/design-system` or `@uranus-workspace/blocks`.

## Roadmap

1. ✅ Ingest the 2026 branding manual → cosmic palette + Lexend Exa / Poppins wired up.
2. ✅ Install the full shadcn/ui primitive catalog (54 components).
3. Author curated MDX pages for every primitive (anatomy, do/don't, a11y).
4. Grow the blocks catalog (AppShell, DataTable, Hero, FeatureGrid, StatCard, CTASection).
5. Turn on Chromatic for visual regression.

## Contributing

- Every new component lands with tests (`*.test.tsx`), a story (`*.stories.tsx`), and an MDX page under `apps/docs/content/docs/`.
- Run `pnpm lint && pnpm typecheck && pnpm test` before opening a PR.
- Use `pnpm changeset` to describe user-facing changes.
