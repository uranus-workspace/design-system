## Manual de marca

O **Manual de Marca Uranus 2026** é a fonte da verdade para a identidade visual. Todo token em `@uranus-workspace/tokens` remete a um valor do manual — a paleta cósmica (Azul Profundo, Azul Marinho, Azul Turquesa, Lilás Claro) e a tipografia Poppins vivem em `packages/tokens/src/colors.ts` e `typography.ts`.

O site vivo da marca é [uranus.com.br](https://uranus.com.br/). Quando o time de marca atualiza o manual, os valores fluem em uma única direção: `packages/tokens/src/*.ts` → CSS gerado → todo app e componente.

## Changelog

Releases são automatizados via [semantic-release](https://semantic-release.gitbook.io/). Commits convencionais (`feat:`, `fix:`, `perf:`, ou algum com `BREAKING CHANGE:`) tocando um pacote publicável disparam o pipeline de release.

## Roadmap

1. ✅ Ingerir o manual de marca 2026 → paleta cósmica + tipografia Poppins conectadas.
2. ✅ Instalar o catálogo completo de primitivos shadcn/ui (54 componentes).
3. ✅ Publicar no npm público como `@uranus-workspace/*`.
4. ✅ Catálogo de blocos para dashboards e painéis (chrome, auth, dados, forms, feedback, growth, marketing) — 30+ blocos.
5. ✅ Pacote `@uranus-workspace/ai` — chat, composer, streaming, áudio, anexos, raciocínio, planejamento e pesquisa, sobre o Vercel AI SDK como peer.
6. Escrever páginas MDX curadas para cada primitivo (anatomia, do/don't, a11y).
7. Adapters de auth opinativos (`@uranus-workspace/blocks-auth-clerk`, `*-next-auth`) sobre os blocos apresentacionais.
8. Ligar o Chromatic para regressão visual.

## Contribuindo

- Todo componente novo chega com testes (`*.test.tsx`), uma story (`*.stories.tsx`) e uma página MDX sob `apps/docs/content/docs/`.
- Rode `pnpm lint && pnpm typecheck && pnpm test` antes de abrir um PR.
- Use commits convencionais para descrever mudanças visíveis pro usuário final.
